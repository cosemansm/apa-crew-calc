# Timesheet Feature Design Spec

**Date:** 2026-04-09
**Status:** Approved

---

## Overview

Add a Timesheet document view to the existing `/invoices` page. Timesheets are a free feature — no pro lock. Invoices remain unchanged but become secondary (slightly de-emphasised), nudging pro users toward bookkeeping integrations instead.

---

## Page Structure

The `/invoices` route currently lands on the invoice builder. After this change it will default to the **Timesheet** view, with Invoice accessible via a secondary toggle/button (marked as Pro).

### Tab / Toggle layout

```
[ Timesheet ]  [ Invoice ↑ Pro ]
```

- Timesheet tab: default, always accessible
- Invoice tab: accessible only to Pro users; shows ProLockOverlay for free users (existing component)
- Bookkeeping integrations (Xero, FreeAgent, QuickBooks) remain in the Invoice view, promoted as the preferred route for Pro users

---

## Timesheet Document Design

### Header

- **Top-left:** CrewDock logo (bolt SVG on #FFD528 square) + "CrewDock" wordmark
- **Below logo:** User's display name (from `user_settings.company_name` or auth name)
- **Below name:** Role pills — one pill per unique `role_name` across selected days (e.g. `Gaffer`, `Lighting Technician`)
- **Top-right:** Label "TIMESHEET" + project name, client name (dash if empty), and date period (`DD Mon – DD Mon YYYY`)
- **Bottom border:** 3px solid #FFD528 accent line
- No VAT number, no reference number

### Day Sections

Each selected project day renders as a section with:

**Day header (single line):**
```
[ MON ]   7 April 2026  ·  Call 07:00  ·  Wrap 19:30          Day total  £648.00
```
- Yellow badge: day of week abbreviation (Mon/Tue/Wed etc.)
- Date: bold
- `Call` label light/small, time bold — same for `Wrap`
- Day total right-aligned
- Background: slightly off-white (`#fafaf8`)

**Line item rows** (inside the day section):

Grouped into sections with a small uppercase group label:

1. **Base Pay**
   - First row: `{Day Type} — {Role}` e.g. `Basic Working Day — Gaffer` or `Continuous Working Day — Lighting Technician`
   - Columns: description · hours/quantity · rate · amount
   - Subsequent rows: overtime tiers with time range and multiplier tag (e.g. `×1.5`, `×2.0`)

2. **Penalties & Allowances** (only if present)
   - Early call penalty with `S.2.3` tag
   - TOC penalty with `TOC` tag and previous wrap/call context
   - Travel pay (hours + rate)
   - Mileage (miles + rate)

3. **Expenses** (only if present)
   - Expense line items from `expenses_notes` / `expenses_amount`

4. **Day subtotal row** — right-aligned, separated by a top border

### Footer

- Left: "Generated by CrewDock · APA Rates 2025" + "This timesheet is not a VAT invoice."
- Right: Grand total amount (large, bold, charcoal)
- No VAT calculation shown

## Data Mapping

| Timesheet field | Source |
|---|---|
| User name | `user_settings.company_name` or Supabase auth display name |
| Role pills | Unique `role_name` values across selected `project_days` |
| Project name | `projects.name` |
| Client name | `projects.client_name` (dash if null/empty) |
| Period | Min/max `work_date` across selected days |
| Day of week | Derived from `work_date` |
| Call / Wrap time | `project_days.call_time` / `project_days.wrap_time` |
| Day type label | Human-readable from `project_days.day_type` enum |
| Base pay line | Day type label + role name + `grand_total` context |
| Line items | `result_json.lineItems` |
| Penalties | `result_json.penalties` |
| Travel pay | `result_json.travelPay` |
| Mileage | `result_json.mileage` / `result_json.mileageMiles` |
| Expenses | `project_days.expenses_amount` / `project_days.expenses_notes` |
| Day subtotal | `project_days.grand_total` |
| Grand total | Sum of all selected `project_days.grand_total` |

### Day type label mapping

| `day_type` value | Display label |
|---|---|
| `basic_working` | Basic Working Day |
| `continuous_working` | Continuous Working Day |
| `prep` | Prep Day |
| `recce` | Recce Day |
| `build_strike` | Build / Strike Day |
| `pre_light` | Pre-Light Day |
| `rest` | Rest Day |
| `travel` | Travel Day |

## Actions

Buttons rendered below the document (app UI, not part of PDF):

- **Download PDF** (primary, yellow) — generates PDF via html2canvas + jsPDF, same approach as existing invoice PDF export
- **Switch to Invoice** (ghost, secondary) — switches to invoice tab; shows Pro lock for free users

---

## Pro / Free Gating

| Feature | Free | Pro |
|---|---|---|
| Timesheet view | ✅ | ✅ |
| Download Timesheet PDF | ✅ | ✅ |
| Invoice view | ❌ (ProLockOverlay) | ✅ |
| Email invoice | ❌ | ✅ |
| Bookkeeping export | ❌ | ✅ |

---

## Project / Day Selection

Timesheet reuses the existing project + day selection UI from InvoicePage:
- Project picker dropdown
- Multi-day selector (checkboxes per day)
- Days sorted chronologically

---

## PDF Generation

- Same 2-pass approach as existing invoice: html2canvas at 2× scale for download quality
- PDF is A4, portrait
- The timesheet document HTML is rendered off-screen and captured
- Filename: `timesheet-{project-name}-{date}.pdf`

---

## What Does NOT Change

- InvoicePage logic, data fetching, and PDF/email flows are untouched
- Bookkeeping integrations (Xero, FreeAgent, QuickBooks) are untouched
- ProLockOverlay component is reused as-is
- Existing routes unchanged (`/invoices` still the route)
